# 大学内ロッカー管理システム

本システムは、大学内のロッカー利用者を公平かつ適正に選定・管理するための自動化ツールです。
Googleフォームから出力された申請データを読み込み、不正チェック（バリデーション）、学生証の目視照合の補助、抽選処理、および結果のPDF出力までを一貫して行います。

---

## 1. 実行環境

本システムは、以下のいずれかの環境で実行できます。

### A. 実行ファイル（.exe）を利用する場合（推奨）
Python環境がないPCでも実行可能です。
* **OS**: Windows 10 / 11
* **必要なもの**: ビルド済みの `main.exe` および、所定のデータフォルダ構造

### B. Python環境を利用する場合（開発・保守用）
* **Pythonバージョン**: Python 3.9 以上推奨
* **依存ライブラリ**:
  * `pandas` (データ処理用)
  * `reportlab` (PDF出力用)
* **インストールコマンド**:
  ```bash
  pip install pandas reportlab

## 2. フォルダ構造
システムを正しく動作させるため、実行ファイル（または main.py）と同じ階層に以下のフォルダ構造を維持してください。

```
locker_system/
├── main.exe                  # 実行ファイル（Python実行の場合は src/main.py）
├── README.md                 # 本ドキュメント
│
├── data/                     # システムが自動更新する永続データ
│   ├── locker_master.csv     # ロッカーの現在の割り当て状態
│   └── winner_history.csv    # 今年度の当選者履歴
│
├── input/                    # 【手動配置】抽選期間ごとの入力データ
│   └── term1/                # 期間名（例: term1, term2...）
│       ├── applicants.csv    # 申請者用フォームCSV
│       └── partners.csv      # 共同利用者用フォームCSV
│
└── output/                   # システムの出力先
    └── term1/
        ├── log/              # バリデーション結果（手動チェック用）
        │   ├── valid_2F.csv ～ valid_6F.csv
        │   ├── invalid_app.csv
        │   └── invalid_par.csv
        ├── result.csv        # 今回の抽選結果（履歴用）
        └── locker_result.pdf # 掲示用PDF
```
## 3. 業務フローと実行手順
抽選業務は、毎週以下の手順（Step 1 〜 Step 4）に沿って実施します。

### Step 1: 入力データの配置
Googleフォームから「申請者用」と「共同利用者用」の回答CSVをダウンロードします。

input/ フォルダ内に今回の期間用フォルダ（例：term1）を作成します。

ダウンロードしたCSVをそれぞれ `applicants.csv`、`partners.csv` にリネームし、作成した期間フォルダ内に配置します。

### Step 2: バリデーション（自動チェック）の実行
`main.exe` をダブルクリックして起動します。

メニューから 1（バリデーション処理）を入力します。

対象期間（例：term1）を入力して実行します。

完了後、`output/term1/log/` フォルダ内に有効データと無効データが振り分けられて出力されます。

### Step 3: 手動チェック（学生証写真の照合）
`output/term1/log/` 内の `valid_2F.csv` 〜 `valid_6F.csv` をExcel等で開きます。

記載されている「学生証写真」のURLを開き、申請者（および共同利用者）の学生証を目視で確認します。

問題なければ、ファイルの一番右にある 学生証照合(TRUE/FALSE) 列に TRUE を入力します（不備・不正があれば FALSE または空欄にします）。

上書き保存してExcelを閉じます（※必ずCSV形式のまま保存してください）。

### Step 4: 抽選処理とPDF出力の実行
再度 `main.exe` を起動し、メニューから 2（抽選処理）を入力します。

対象期間（例：term1）を入力して実行します。

手動チェックで TRUE とされた応募者の中から、空き枠に応じた抽選が自動で行われます。

`output/term1/` フォルダ内に、当選結果の `result.csv` と、掲示用の `locker_result.pdf` が出力されます。

## 4. 実行ファイル（.exe）の作成手順
ソースコードを修正した際などは、以下の手順で新しい実行ファイル（exe）を作成します。

Python環境があるPCで、コマンドプロンプト（またはターミナル）を開きます。

必要なビルドツールをインストールします。

```
pip install pyinstaller pandas reportlab
```
locker_system フォルダに移動し、以下のコマンドを実行します。

```
pyinstaller --onefile --name "main" src/main.py
```
処理が完了すると、dist/ フォルダの中に main.exe が生成されます。

生成された main.exe を、元の locker_system/ フォルダ直下に移動・上書きしてください。
（※ build/ フォルダと main.spec ファイルは不要なので削除して構いません）

## 5. 注意事項・保守について
年度更新（リセット）の運用:
ロッカーの利用は年度単位です。新年度の受付を開始する前に、必ず以下のリセット作業を行ってください。

data/winner_history.csv の中身を空にする（ヘッダー行だけ残すか、ファイルごと削除する）。

data/locker_master.csv の「割り当て状態」列をすべて空にする（ファイルごと削除すると、次回実行時に空の状態で自動再生成されます）。

多重応募の仕様:
同一人物による複数応募は「最新のものを優先」し、異なるペアでの競合は「共同利用者の提出が早いペアを優先」する仕様になっています。システムが自動でS0（有効）やE4（マッチング済みエラー）に振り分けます。

PDFの表示内容:
個人情報保護のため、抽選結果のPDFには「申請者の学籍番号」のみが記載されます。
